# สรุปรูปแบบเกม Isekai RPG สำหรับการพัฒนา (Unity C# Game Concept)

## 1. ข้อมูลพื้นฐานและฉากหลัง (Core Concept & Setting)

| หัวข้อ | รายละเอียด |
| :--- | :--- |
| **ประเภทเกม** | Isekai Fantasy RPG / MMORPG Elements (Open World, Trading, High Risk/Reward PK) |
| **ฉากหลัง** | ผู้เล่นถูกอัญเชิญมาต่างโลกหลังเกิดอุบัติเหตุ เริ่มต้นที่หมู่บ้านเล็กๆ |
| **เป้าหมายหลัก** | ลงทะเบียนเป็นนักผจญภัย, ทำเควสเนื้อเรื่อง, และใช้ชีวิตอย่างอิสระ (Exploration, Farming, Trading, PvP/PK) |

## 2. การสร้างตัวละคร (Character Creation)

### 2.1 การเลือกตัวละคร

* **เผ่าพันธุ์ (Race):** มนุษย์ (Human), เอลฟ์ (Elf), ก็อบลิน (Goblin), สัตว์ (Animal), ปีศาจ (Demon), เทพ (God)
* **เพศ (Gender):** เลือกได้ (Male/Female)
* **รูปลักษณ์:** รูปลักษณ์ตายตัวตามเผ่าพันธุ์ ไม่สามารถตกแต่งหน้าตาได้
* **อาวุธเริ่มต้น:** เลือกประเภทอาวุธหลัก (ดาบ, ธนู, เวท, หอก)

## 3. ระบบอาวุธและสกิล (Weapon & Skill System)

### 3.1 ระบบอาวุธ

* **ข้อจำกัด:** เมื่อเลือกประเภทอาวุธแล้วจะใช้ได้แค่ประเภทนั้นเท่านั้น
* **อาวุธย่อย/อาชีพ:** มีอาวุธย่อยที่สอดคล้องกับอาชีพ (เช่น ดาบใหญ่, ดาบโล่, ดาบคู่, มีดสั้น(นักฆ่า) สำหรับประเภทดาบ)
* **ประสิทธิภาพ:** การติดตั้งอาวุธต้องสอดคล้องกับสกิลที่ใช้ หากไม่ซัพพอร์ต ประสิทธิภาพของสกิลจะลดลง
* **การเปลี่ยนประเภท:** ต้องซื้อ **การ์ดเปลี่ยนประเภทอาวุธ** (เงินเติม/Premium Currency เท่านั้น)

### 3.2 ระบบสกิล

* **1. สกิลติดอาวุธ (Passive/Weapon Skill):** ไม่สามารถกดใช้ได้ ใช้สำหรับเพิ่มความสามารถของอาวุธ/ผู้เล่น
* **2. สกิลแบบร่าย (Active/Casting Skill):** ต้องกดใช้เพื่อแสดงผล มีผลกระทบต่ออาวุธที่สวมใส่ (ซัพพอร์ต = ประสิทธิภาพเพิ่ม / ไม่ซัพพอร์ต = ประสิทธิภาพลด)
* **การเรียนรู้/อัปเกรด:** ใช้วิธี **สะสมไอเทมเฉพาะ** เพื่อแลกซื้อสกิล และสะสมไอเทมเพื่ออัพสเตตัสสกิล (ไม่ใช้ระบบแต้มอัพสกิล)

## 4. สัตว์อสูร (Spirit Beast / Pet System)

* **หน้าที่:** เพิ่ม Buff ให้แก่ผู้เล่น
* **การอัปเกรด:** ใช้ **ไอเทมเฉพาะ** ในการอัปเกรด
* **การได้รับ:** ไข่ฟักอสูรดร็อปใน **Red maps** เท่านั้น (สุ่มสัตว์อสูร)
* **ระดับ:** มีระดับ (F, E, D, B, A, S, SR, SSR)

## 5. ระบบกระเป๋าและบทลงโทษการตาย (Inventory & Death Penalty)

### 5.1 ประเภทกระเป๋า

* **1. กระเป๋าส่วนเซฟตี้ (Safety Bag):**
    * ช่องเก็บจำกัด ไม่สามารถอัปเกรดได้
    * **ไอเทมไม่ตกเมื่อตาย**
* **2. กระเป๋าเก็บไอเท็ม (Item Bag):**
    * สามารถอัปเกรดเพิ่มช่องได้ด้วยการสะสมไอเทม
    * **ไอเทมตกเมื่อตาย** ในพื้นที่เสี่ยง

### 5.2 บทลงโทษการตาย (Death Penalty)

* **ทุกการตาย:** ถูกลบค่า EXP
* **Red maps:** ไอเทมทั้งหมดในกระเป๋าเก็บไอเท็มตก **100%** (ไม่รวมเครื่องสวมใส่และกระเป๋าเซฟตี้)
* **ดันเจี้ยน:** ไอเทมประเภทวัตถุดิบในกระเป๋าเก็บไอเท็มตก **30%** (ไม่รวมเครื่องสวมใส่และกระเป๋าเซฟตี้)

## 6. ระบบเศรษฐกิจและการแลกเปลี่ยน (Economy & Trading)

### 6.1 ระบบฟาร์ม, เลี้ยงสัตว์, ขุดเหมือง (Life Skills)

* **อุปกรณ์เริ่มต้น:** จอบ, บัวรดน้ำ, เคียว, ขวาน, ที่ขุดหิน
* **การพัฒนา:** สะสมไอเท็มเพื่อเพิ่มประสิทธิภาพของอุปกรณ์ Life Skills

### 6.2 การซื้อขายและแลกเปลี่ยน

| รูปแบบ | เงื่อนไข | ค่าธรรมเนียม/ข้อจำกัด |
| :--- | :--- | :--- |
| **ขายของให้ NPC** | ขายไอเทม/ของคราฟท์ให้ร้านค้าในเกม | ไม่มี |
| **1. ระบบตั้งร้านขาย (Player Shop)** | ต้องอยู่ในเมืองที่อนุญาต **ต้องออนไลน์และไม่สามารถขยับได้** ผู้ซื้อคลิกที่ตัวผู้ขาย | มีค่าตั้งร้านเป็นเงินในเกม |
| **2. ระบบฝากขาย (Consignment)** | ต้องซื้อ **การ์ดฝากขาย** (เงินเติม) ขายในตลาดกลาง **สามารถทำกิจกรรมอื่นได้ปกติขณะขาย** | การ์ดมีระยะเวลาและจำนวนชิ้นสินค้าที่กำหนด |
| **ระบบแลกของระหว่างผู้เล่น (Direct Trade)** | คลิกที่ตัวผู้เล่นอื่นและเลือกเมนูแลกไอเทม | มีค่าแลกเป็นเงินในเกม |

## 7. ระบบการต่อสู้ระหว่างผู้เล่น (PVP & PK System)

### 7.1 ระบบ PVP (Arena)

* **สถานที่:** จุดสเตเดี้ยม (มีเมนูสร้างห้อง)
* **การสร้างห้อง:**
    * **1 vs 1:** สร้างห้องได้ด้วย **เงินในเกม**
    * **Team vs Team:** สร้างห้องได้ด้วย **เงินเติม**
* **รูปแบบผู้ชม:** เข้าชมได้ฟรี

### 7.2 ระบบ PK (Player Killing)

* ผู้เล่นสามารถฆ่ากันได้ในพื้นที่ทั่วไป **ยกเว้นพื้นที่ห้าม PK**

## 8. โฟลว์เริ่มต้นของเกม (Initial Game Flow / Tutorial)

1.  **ฉากอนิเมชั่น:** ผู้เล่นเกิดอุบัติเหตุ
2.  **การสร้างตัวละคร:** สร้างตัวละครเสร็จ
3.  **เริ่มต้น:** ตื่นขึ้นมาบนเตียงในบ้านเล็กๆ ในหมู่บ้าน (หน้าบ้านมีแปลงผักรกร้าง)
4.  **ได้รับไอเท็ม:** ไอเท็มเริ่มต้นเพื่อนำไปซื้ออาวุธเริ่มต้นที่ร้านค้าในหมู่บ้าน
5.  **เควสหลักเริ่มต้น:** คุยกับพ่อค้าเรื่องกิลด์นักผจญภัย
6.  **ลงทะเบียน:** เดินทางไปกิลด์นักผจญภัยเพื่อลงทะเบียน (จบช่วง Tutorial)
7.  **อิสระ:** เล่นได้อย่างอิสระ โดยมีเควสเนื้อเรื่องหลักให้เล่นควบคู่ไป (เช่น ปราบมอนสเตอร์, ทำฟาร์ม, หาเพื่อน, ซื้อขาย)

## 9. ความคิดเห็นและข้อเสนอแนะสำหรับนักพัฒนา

### 9.1 จุดเด่นที่น่าสนใจ

* **ความเสี่ยงสูง-ผลตอบแทนสูง (High Risk, High Reward):** ระบบ Item Drop 100% ใน Red Maps และการเสีย EXP เมื่อตาย สร้างความตึงเครียดและมูลค่าให้กับไอเทม
* **ระบบเศรษฐกิจที่ยั่งยืน:** การซื้อขายที่มีทั้งช่องทาง In-game Money (ตั้งร้าน) และ Premium Money (ฝากขาย) ช่วยสร้างมิติการค้าขายที่หลากหลาย
* **การเชื่อมโยงการเล่น:** การอัปเกรดสกิลที่ต้องพึ่งพา **ไอเทมเฉพาะ** และ Life Skills จะบังคับให้ผู้เล่นมีส่วนร่วมกับทุกระบบในเกม

### 9.2 ข้อควรพิจารณาเพิ่มเติมสำหรับทีมพัฒนา (Unity C# Implementation)

* **ความสมดุลของ Red Maps/PK:** ให้ความสำคัญกับการจัดการ Server Stability, Lag, และ Anti-Cheat ในพื้นที่ Full Loot PK เป็นอันดับแรก เพื่อสร้างประสบการณ์ที่ยุติธรรม
* **การออกแบบ UI/UX:** ต้องออกแบบส่วนติดต่อผู้ใช้ของกระเป๋า (Inventory) ให้ชัดเจนมากว่าไอเทมไหนอยู่ใน **Safety Bag** และไอเทมไหนอยู่ใน **Item Bag** เพื่อป้องกันความเข้าใจผิดของผู้เล่นเมื่อเสียชีวิต
* **กลไกสกิล:** กำหนดรายละเอียดของผลกระทบเมื่ออาวุธ **ไม่ซัพพอร์ต** สกิลให้ชัดเจน (เช่น ลดพลังโจมตีลงเป็นเปอร์เซ็นต์, เพิ่มคูลดาวน์, หรือใช้มานามากขึ้น)
